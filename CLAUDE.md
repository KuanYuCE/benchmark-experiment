# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Build & Run

```bash
# Main benchmark (Euclidean vs Binary vs Hybrid/GMP)
gcc -O2 -o gcd_bench gcd_bench.c -lgmp

# Division cost micro-benchmark (64-bit vs 128-bit modulo raw cost)
gcc -O2 -o div_cost div_cost.c

# Run benchmark — takes ~3 minutes, writes results.csv
./gcd_bench

# Run micro-benchmark — takes ~30 seconds
./div_cost
```

Requires `libgmp-dev` (`sudo apt-get install -y libgmp-dev`).

## Architecture

Two independent C programs; no shared headers or build system.

**`gcd_bench.c`** — the main experiment, two sections:

- **Section 1 (Native)**: compares `euc64`/`bin64` (8–64-bit) and `euc128`/`bin128` (128-bit) using fixed-width types. Each tier runs 10M pairs. Two passes per tier: a clean timing pass and a separate instrumented ops-counting pass, so `++count` overhead never distorts timing.
- **Section 2 (GMP)**: compares `mpz_gcd` vs `hybrid_gcd` across 64-bit–2048-bit. `hybrid_gcd` dispatches: `euc64` if both operands fit in one limb, `bin128` if both fit in two limbs, otherwise `mpz_gcd`. Pairs are pre-generated with `gmp_randinit_mt` before timing starts.

Key design decisions:
- `volatile sink` XOR'd with every result prevents dead-code elimination.
- RNG is xorshift64 with fixed `SEED=42` for reproducibility.
- `euc128` uses `%` on `__uint128_t` → triggers `__udivti3` (libgcc software division). `bin128` avoids division entirely (shifts + sub only), making it safe in `-nostdlib` environments like xv6 kernel.
- `ctz128` is a manual two-step `__builtin_ctzll` because `__builtin_ctzll` doesn't accept `__uint128_t`.

**`div_cost.c`** — isolated micro-benchmark measuring raw ns/op for a single 64-bit vs 128-bit modulo, used to confirm the causal "expensive division → Binary wins" finding.

**Output files** (not source, regenerated by running the benchmarks):
- `results.csv` — structured data from `gcd_bench`
- `visualize.html` — Chart.js interactive charts (needs network for CDN); open in browser
- `report.md` / `report.html` — written experiment reports with environment spec and analysis

## Key Experimental Findings

- 8–64-bit: Euclidean always wins (`t_ratio` 0.73–0.99). `ops_ratio` is stable at ~0.83 across all tiers — Euclidean does ~17% fewer iterations regardless of bit width.
- 128-bit: Binary wins (`t_ratio` 1.016–1.063). Crossover caused by `__udivti3` software emulation (4.6 ns/op vs 1.7 ns/op for hardware div).
- GMP 64/128-bit: `mpz_gcd` beats `hybrid_gcd` — GMP already has internal fast paths (`mpn_gcd_1` for single-limb) that are cheaper than the `mpz_to_u128` / `u128_to_mpz` conversion overhead in hybrid.
- GMP 256-bit+: hybrid ≈ mpz_gcd (within 1% noise); the two `mpz_size()` checks in hybrid are negligible.

**Corrected hypothesis**: "division becomes expensive → Binary wins", not "larger numbers → Binary wins".
