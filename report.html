演算法

Euclidean GCD

gcd(a, b) = gcd(b, a % b)

關鍵成本：modulo 運算，直接對應 CPU 除法指令。

Binary GCD（Stein's）

用 >> 和 -= 取代 %

關鍵成本：位移與減法，在 64-bit 硬體上均為單週期。

分析與結論

ops_ratio 全程穩定在 ~0.83： 無論幾 bit，Euclidean 永遠比 Binary 少做約 17% 的 iteration。 演算法層次上，Euclidean 本來就更精簡。

64-bit 以下 Euclidean 贏： 硬體除法器快（1.7 ns/op），iteration 少 + 每次 op 便宜 = 雙重優勢。

128-bit 翻轉： __uint128_t % __uint128_t 觸發 libgcc 的 __udivti3 軟體模擬（4.6 ns/op）。 此時 Euclidean 每次 mod 成本暴增，Binary 的位移操作仍是單週期，最終翻盤。

修正後的假說

❌ 原始假說（不正確）

「數字越大 → Binary 贏」

✅ 修正假說（正確）

「除法指令越貴 → Binary 贏」

在現代 x86-64 硬體上，64-bit 硬體除法器快到讓 Euclidean 在整個原生寬度都保有優勢。 交叉點出現在「溢出硬體除法器寬度」的時候，不是「數字變大」本身。
